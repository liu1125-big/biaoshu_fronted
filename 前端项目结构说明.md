## 三、功能模块详解

### 3.1 technical-plan（标书生成）— 最核心模块

```

├── types.ts                    # 全部类型定义（State、Step、File、Config 等）

│   ├── GlobalFactsPage.tsx     # 步骤4：全局事实/企业信息
│   └── ContentEditPage.tsx     # 步骤5：章节内容生成/编辑
```

**各文件作用：**

| 文件 | 行数 | 作用 |

| `DocumentAnalysisPage.tsx` | ~437 | 步骤1：上传招标文件/原方案，显示解析后的 Markdown，支持标段选择 |

## 五、数据流概览

```
用户操作 → React 组件
              ↓
         自定义 Hook（状态管理 + 业务逻辑）
              ↓
         apiClient.knowledgeBase / apiClient.technicalPlan / apiClient.tasks
              ↓
         axios HTTP 请求 / EventSource (SSE)
              ↓
         ENDPOINTS 常量（URL 拼接自 VITE_API_BASE）
              ↓
         后端 API ← 当前不存在，Vite 返回 index.html（SPA fallback）
```

**向后端发送请求的方式有两种：**
1. **普通 HTTP**（axios）：GET/POST/PUT/DELETE → 用于数据 CRUD
2. **SSE (Server-Sent Events)**（EventSource）：GET 长连接 → 用于实时任务进度推送

## 七、接入后端时需要修改的文件

### 必须修改

| 文件 | 改动 |
|------|------|
| `.env` | 设置 `VITE_API_BASE=http://your-backend:port` |
| `vite.config.ts` | 如需代理 API 请求，配置 `server.proxy` |

### 通常不需要修改

| 文件 | 原因 |
|------|------|
| `endpoints.ts` | 端点路径已按 RESTful 约定定义好 |
| `apiClient.ts` | 已实现完整 HTTP 调用，SSE 也已配置 |
| 所有 feature 文件 | 组件和 Hook 都通过 apiClient 间接调用后端，不直接访问端点 |

### 可能需要调整

| 文件 | 场景 |
|------|------|
| `endpoints.ts` | 后端 API 路径与预期不一致时 |
| `apiClient.ts` 中对应方法 | 请求/响应格式与后端不一致时 |
| feature 中的 types.ts | 后端返回的数据结构与前端类型不匹配时 |
| feature 中的 hooks | 业务逻辑需要适配后端行为时 |
